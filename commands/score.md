# /score - 품질 점수 확인

현재 코드의 품질 점수를 보여줍니다. (Advanced: `/trinity`)

## 사용법

```
/score              # 전체 점수 확인
/score "변경사항"   # 특정 변경에 대한 점수
```

## 점수 구성

| 항목 | 비중 | 의미 |
|------|------|------|
| 정확성 | 35% | 코드가 올바른가? |
| 안전성 | 35% | 문제가 없는가? |
| 가독성 | 20% | 읽기 쉬운가? |
| 편의성 | 8% | 쓰기 편한가? |
| 지속성 | 2% | 오래 유지되는가? |

## 결과 예시

```
품질 점수: 92/100

✅ 정확성: 95점 - 타입 안전
✅ 안전성: 90점 - 테스트 통과
✅ 가독성: 88점 - 린트 통과
✅ 편의성: 92점 - 사용 편리
✅ 지속성: 90점 - 문서화 양호

→ 자동 실행 가능 (90점 이상)
```

## 점수별 행동

| 점수 | 상태 | 다음 행동 |
|------|------|----------|
| 90+ | 안전 | 바로 진행 |
| 70-89 | 주의 | 확인 후 진행 |
| 70 미만 | 위험 | 수정 필요 |
